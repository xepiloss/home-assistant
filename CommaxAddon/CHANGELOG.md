# Changelog

## 1.1.3 - 2026-04-25

### Added

- 잘못 정렬된 TCP 스트림 내부에서 checksum이 유효한 상태 프레임을 복원해 기존 상태/ACK 처리 흐름으로 전달합니다.
- 복원된 상태 프레임은 애드온 로그에 `패킷 복원 성공`으로 남기고, unknown 패킷 캡처가 켜진 경우 `recovered_state_frame`으로도 기록합니다.

### Changed

- ACK 대기 중인 같은 장치 명령은 즉시 추가 전송하지 않고 마지막 명령만 보류했다가 ACK 또는 실패 처리 후 전송하도록 했습니다.
- MQTT 명령 수신 시 Home Assistant 상태를 먼저 갱신하지 않고, 실제 월패드 ACK/상태 프레임 수신값으로만 상태가 바뀌도록 했습니다.
- 장시간 값이 변하지 않은 고빈도 주기 프레임은 known stable 프레임으로 메모하고 unknown 패킷 캡처에서 제외합니다.
- 생활정보 `8F` 프레임은 완전 동일 반복값만 제외하고, 값이 변한 프레임은 날씨/미세먼지 분석을 위해 계속 수집되도록 유지했습니다.

### Fixed

- 주차 차량번호 프레임에서 `6` 같은 단일 문자 조각은 무시하되, `1234`, `AR:1`, `A1`처럼 온전하게 해석되는 차량 표기는 발행하도록 오탐 방어 조건을 조정했습니다.

## 1.1.2 - 2026-04-25

### Added

- MQTT, 메인 EW11, 검침 EW11 연결 상태를 Home Assistant 진단 엔티티로 발행합니다.
- 메인/검침 EW11 마지막 수신 시간을 진단 센서로 추가하되, 활동 로그 증가를 피하기 위해 기본 비활성 상태로 등록합니다.
- `24 01` 생활정보 온도 프레임도 알 수 없는 패킷 수집 파일에 함께 남기도록 했습니다.

### Changed

- 월패드 시간 센서를 진단 엔티티 기본 비활성 상태로 재발행되도록 discovery 버전을 갱신했습니다.
- MQTT 연결 상태에 LWT를 설정해 비정상 연결 종료 시 `disconnected` 상태가 남도록 했습니다.
- 실시간 수도/온수/가스 사용량의 `m³/h` 단위가 HA에서 유효하게 처리되도록 device class를 `volume_flow_rate`로 변경했습니다.

### Fixed

- 실시간 검침 센서에 `state_class: measurement`를 추가해 `0`도 정상 측정값으로 표시되도록 정리했습니다.
- 기존 상태 파일에 검침 discovery 완료 기록이 있어도 실시간 검침 sensor discovery가 다시 발행되도록 discovery 버전 키를 갱신했습니다.

## 1.1.1 - 2026-04-25

### Added

- 월패드 시간 센서를 추가하고 사람이 읽을 수 있는 텍스트 상태로 발행합니다.
- 알 수 없는 패킷 수집 옵션을 추가했습니다. 같은 패킷은 하나로 묶고 수신 시각만 누적합니다.
- 확인된 `0x24` 생활정보 온도 프레임을 `생활정보 온도` 센서로 발행합니다.

### Changed

- 월간 검침 보정값 적용/유지/자동 기준값 설정 로그에 입력값, 현재 누적값, 월초 누적 기준값, 계산식을 남깁니다.
- 월간 검침 월 판단에 월패드 시간을 우선 사용합니다.
- 메인 EW11과 검침 EW11의 `F7` 프레이밍 규칙을 분리했습니다.
- 생활정보 원본 패킷은 HA 센서로 디스커버리하지 않고, 필요 시 패킷 수집 파일로만 추적합니다.

### Fixed

- 월간 검침 보정값이 반복 적용되어 이번달 사용량이 보정값에 고정될 수 있는 문제를 수정했습니다.
- `YYYY-MM` 형식이 아닌 보정 월, 음수 보정값, 현재 누적값보다 큰 보정값은 무시하도록 했습니다.
- 생활정보/시간 센서가 HA Logbook을 과도하게 증가시키지 않도록 발행 빈도와 discovery를 정리했습니다.

## 1.1.0 - 2026-04-25

이번 릴리스는 실사용 안정성과 Home Assistant 사용감을 크게 다듬은 기능 업데이트입니다.

### Added

- EW11 TCP 스트림 프레이밍을 추가해 쪼개지거나 붙어서 들어오는 패킷을 안정적으로 처리합니다.
- 원격 검침 `F7` 가변 길이 프레임을 지원합니다.
- 월간 검침 센서를 추가했습니다.
  - 이번달 전기 사용량
  - 이번달 수도 사용량
  - 이번달 온수 사용량
  - 이번달 난방 사용량
  - 이번달 가스 사용량
- 월 중간 설치 사용자를 위한 월간 사용량 보정 옵션을 추가했습니다.
- 명령 송신/수신 로그에 디바이스 종류, ID, 명령 종류, 응답 시간을 표시합니다.

### Changed

- 같은 디바이스에 빠르게 연속 명령이 들어올 때 최신 명령 중심으로 처리되도록 개선했습니다.
- 조명 밝기 명령 직후 Home Assistant가 보내는 중복 `ON` 명령을 억제합니다.
- 빠른 조작 시 ACK/STATE 매칭 정확도를 높여 불필요한 재전송 가능성을 줄였습니다.
- 명령 재전송 시점을 실제 송신 기준으로 조정해 빠른 연속 제어 안정성을 개선했습니다.
- 종료 처리에서 상태 저장과 availability publish가 더 안정적으로 수행되도록 정리했습니다.
- 검침, 공기질, 주차 센서에 값 성격에 맞는 아이콘을 지정했습니다.

### Fixed

- TCP 조각 수신으로 검침 프레임이 유실되거나 unknown packet으로 분리되던 문제를 수정했습니다.
- 짧은 공기질 패킷이 들어와도 파서가 죽지 않도록 방어 코드를 추가했습니다.
- 검침 장비가 분리된 환경에서도 원격 검침 프레임이 정상 파싱되도록 개선했습니다.

### Notes

- 월간 검침 센서는 애드온이 전체 누적값을 기준으로 내부 월초 기준값을 저장해 계산합니다.
- 월 중간에 설치했다면 애드온 옵션의 `monthly_*_usage` 값에 현재까지의 이번 달 사용량을 입력하면 됩니다.
- 기존 사용자도 다음 검침/공기질/주차 패킷 수신 시 discovery가 갱신되어 새 아이콘을 받을 수 있습니다.

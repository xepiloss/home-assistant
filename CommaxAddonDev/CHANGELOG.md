# Changelog

## 1.1.12-dev - 2026-04-25

### Fixed

- 기존 상태 파일에 검침 discovery 완료 기록이 있어도 실시간 검침 sensor discovery가 다시 발행되도록 discovery 버전 키를 갱신했습니다.

## 1.1.11-dev - 2026-04-25

### Fixed

- 실시간 수도/온수/가스 사용량의 `m³/h` 단위가 HA에서 유효하게 처리되도록 device class를 `volume_flow_rate`로 변경했습니다.
- 실시간 검침 센서에 `state_class: measurement`를 추가해 `0`도 정상 측정값으로 표시되도록 정리했습니다.

## 1.1.10-dev - 2026-04-25

### Changed

- `24 01` 생활정보 온도 프레임도 알 수 없는 패킷 수집 파일에 함께 남기도록 했습니다.
- 온도 값으로 해석된 프레임이라도 `80/85` 등 아직 의미가 확정되지 않은 바이트를 분석할 수 있도록 동일 패킷 단위로 수신 시각을 누적합니다.

## 1.1.9-dev - 2026-04-25

### Changed

- 메인 EW11과 검침 EW11의 `F7` 프레이밍 규칙을 분리해 메인 EW11의 `F7` 프레임이 검침 가변 길이 규칙으로 잘리지 않도록 했습니다.
- `0x24 01 01 20 ?? TT 00` 생활정보 온도 프레임을 `생활정보 온도` 센서로 발행하도록 했습니다.
- 생활정보 온도 프레임의 `80/85` 값은 의미를 확정하지 않고 `unknown_code` 속성으로 함께 남기도록 했습니다.

## 1.1.8-dev - 2026-04-25

### Changed

- 월간 검침 보정 월이 `YYYY-MM` 형식이 아니거나 월 범위가 아니면 보정값을 무시하도록 했습니다.
- 월간 검침 보정값이 음수이거나 현재 누적값보다 크면 월초 누적 기준값에 적용하지 않도록 했습니다.
- 월간 검침 보정 적용 로그에 현재 누적값, 계산된 월초 누적 기준값, 이번달 사용량 계산식을 함께 남기도록 했습니다.

## 1.1.7-dev - 2026-04-25

### Changed

- 월패드 시간 센서 상태값에서 `시간:` 접두어를 제거했습니다.
- 애드온 시작 시 월간 검침 보정값 입력 여부와 적용 예정 값을 로그로 남기도록 했습니다.
- 월간 검침 보정값이 적용될 때 입력 보정값, 현재 누적값, 계산된 월초 누적 기준값을 로그로 남기도록 했습니다.
- 월간 검침 자동 기준값을 설정할 때 월패드/시스템 기준과 월 변경 내역을 로그로 남기도록 했습니다.

## 1.1.6-dev - 2026-04-25

### Changed

- HA Recorder/Logbook 증가를 줄이기 위해 월패드 시간 센서 상태를 초 단위가 아닌 분 단위로만 발행합니다.
- 월패드 시간은 내부 월간 검침 기준값으로는 계속 초 단위 최신 값을 유지하고, HA 센서 상태만 `시간: YYYY-MM-DD HH:mm` 형식으로 제한합니다.

## 1.1.5-dev - 2026-04-25

### Fixed

- 기존에 등록된 월패드 시간 센서의 `timestamp` Discovery 설정이 남아 있어 상대 시간처럼 보이던 문제를 수정했습니다.
- 월패드 시간 Discovery를 강제로 재발행하고, 상태값 앞에 `시간:` 접두어를 붙여 텍스트 센서로 표시되도록 했습니다.

## 1.1.4-dev - 2026-04-25

### Fixed

- 월간 검침 보정값이 매 검침 패킷마다 다시 적용되어 이번달 사용량이 보정값에 고정되던 문제를 수정했습니다.
- 보정값은 최초 적용 또는 보정 설정 변경 시에만 기준값을 조정하고, 이후에는 누적 사용량 증가분이 이번달 사용량에 반영되도록 했습니다.

## 1.1.3-dev - 2026-04-25

### Changed

Home Assistant에서 Dev 애드온 업데이트가 감지되도록 버전을 올린 개발 배포입니다.

- 동일 패킷 집계, 생활정보 `8F` 캡처, 월패드 시간 문자열 표시, 생활정보 raw Discovery 제거 변경을 HA 업데이트로 받을 수 있게 했습니다.

## 1.1.1-dev - 2026-04-25

패킷 수집과 생활정보 분석을 계속하기 위한 개발용 애드온 배포입니다.
[commax_unknown_packets.jsonl](../../../../../Volumes/share/commax_unknown_packets.jsonl)
### Added

- 개발용 애드온 `Commax Wallpad Dev`를 추가했습니다.
- Dev 애드온 slug를 `commax_wallpad_dev`로 분리했습니다.
- 알 수 없는 패킷 수집 옵션을 추가하고 Dev 애드온에서는 기본 ON으로 설정했습니다.
- 알 수 없는 프레임과 프레이밍 중 버려진 바이트를 `/share/commax_unknown_packets.jsonl`에 JSON Lines 형식으로 저장합니다.
- 동일한 알 수 없는 패킷은 하나의 레코드로 묶고 `seen_at` 수신 시각 배열만 누적하도록 했습니다.
- 생활정보 `8F` 프레임도 값 변동 분석을 위해 `life_info_frame`으로 수집 파일에 저장합니다.
- 알려진 조회/상태성 패킷은 수집 대상에서 제외해 분석 대상 패킷만 남기도록 필터링했습니다.
- `7F` 월패드 시간 패킷을 파싱하고 사람이 읽기 쉬운 문자열 센서로 등록합니다.
- 월간 검침 사용량의 월 판단에 월패드 시간을 우선 사용하도록 개선했습니다.
- `8F` 생활정보 응답은 Home Assistant Discovery 센서로 등록하지 않고 캡처 파일에서만 비교 분석하도록 정리했습니다.

### Notes

- 배포용 `Commax Wallpad`와 Dev 애드온을 같은 EW11에 동시에 연결하지 마세요.
- `8F` 생활정보 패킷의 날씨/기온/미세먼지 매핑은 실제 월패드 표시값과 더 비교한 뒤 확정할 예정입니다.
- 같은 패킷은 중복 본문을 저장하지 않지만 수신 시각은 계속 누적되므로 장기 수집 후에는 필요 시 백업하거나 정리해 주세요.

## 1.1.0 - 2026-04-25

이번 릴리스는 실사용 안정성과 Home Assistant 사용감을 크게 다듬은 기능 업데이트입니다.

### Added

- EW11 TCP 스트림 프레이밍을 추가해 쪼개지거나 붙어서 들어오는 패킷을 안정적으로 처리합니다.
- 원격 검침 `F7` 가변 길이 프레임을 지원합니다.
- 월간 검침 센서를 추가했습니다.
  - 이번달 전기 사용량
  - 이번달 수도 사용량
  - 이번달 온수 사용량
  - 이번달 난방 사용량
  - 이번달 가스 사용량
- 월 중간 설치 사용자를 위한 월간 사용량 보정 옵션을 추가했습니다.
- 명령 송신/수신 로그에 디바이스 종류, ID, 명령 종류, 응답 시간을 표시합니다.

### Changed

- 같은 디바이스에 빠르게 연속 명령이 들어올 때 최신 명령 중심으로 처리되도록 개선했습니다.
- 조명 밝기 명령 직후 Home Assistant가 보내는 중복 `ON` 명령을 억제합니다.
- 빠른 조작 시 ACK/STATE 매칭 정확도를 높여 불필요한 재전송 가능성을 줄였습니다.
- 명령 재전송 시점을 실제 송신 기준으로 조정해 빠른 연속 제어 안정성을 개선했습니다.
- 종료 처리에서 상태 저장과 availability publish가 더 안정적으로 수행되도록 정리했습니다.
- 검침, 공기질, 주차 센서에 값 성격에 맞는 아이콘을 지정했습니다.

### Fixed

- TCP 조각 수신으로 검침 프레임이 유실되거나 unknown packet으로 분리되던 문제를 수정했습니다.
- 짧은 공기질 패킷이 들어와도 파서가 죽지 않도록 방어 코드를 추가했습니다.
- 검침 장비가 분리된 환경에서도 원격 검침 프레임이 정상 파싱되도록 개선했습니다.

### Notes

- 월간 검침 센서는 애드온이 전체 누적값을 기준으로 내부 월초 기준값을 저장해 계산합니다.
- 월 중간에 설치했다면 애드온 옵션의 `monthly_*_usage` 값에 현재까지의 이번 달 사용량을 입력하면 됩니다.
- 기존 사용자도 다음 검침/공기질/주차 패킷 수신 시 discovery가 갱신되어 새 아이콘을 받을 수 있습니다.

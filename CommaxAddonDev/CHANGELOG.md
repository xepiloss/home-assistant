# Changelog

## 1.1.30-dev - 2026-04-26

### Changed

- 검침 센서 Discovery에 `suggested_display_precision`을 추가해 Home Assistant 기본 표시 자릿수를 실제 패킷 정밀도에 맞추도록 했습니다.

## 1.1.29-dev - 2026-04-25

### Fixed

- 비정상 조명 상태 프레임이 `조명 67` 같은 존재하지 않는 조명으로 Discovery 되는 것을 방지했습니다.
- 이미 저장된 비정상 조명 Discovery retained 설정은 애드온 시작 시 자동 정리하도록 했습니다.

## 1.1.28-dev - 2026-04-25

### Added

- `mqtt` 엘리베이터 모드에서 호출 스위치 Discovery와 함께 `commax/ev` 토픽 기반 엘리베이터 1/2 현재층 센서를 자동 Discovery 하도록 추가했습니다.

## 1.1.27-dev - 2026-04-25

### Changed

- Home Assistant 애드온 스토어에서 Dev 빌드가 실험용으로 표시되도록 `stage: experimental` 메타데이터를 추가했습니다.

## 1.1.26-dev - 2026-04-25

### Changed

- 신규 설치 기본 엘리베이터 호출 모드를 `off`로 변경해 사용자가 명시적으로 선택하기 전에는 EV Discovery가 생성되지 않도록 했습니다.
- README에 SOAP/TCP-IP 호출용 MQTT 전달 모드와 RS485 직접 호출 모드 설정 방법을 정리했습니다.
- 검침 누적값 중 수도/가스/온수/난방도 전기와 동일하게 3바이트 BCD 누적값으로 파싱하도록 보정했습니다.

## 1.1.25-dev - 2026-04-25

### Changed

- 엘리베이터 호출 모드에 `off`를 추가해 비활성 선택 시 Discovery 발행과 MQTT/RS485 명령 처리를 하지 않도록 했습니다.
- `off` 모드에서는 기존에 남아 있던 엘리베이터 MQTT Discovery retained 설정도 정리합니다.
- `mqtt` 모드는 SOAP 애드온 연동용으로 애드온 시작 시 엘리베이터 switch Discovery를 즉시 발행합니다.
- `rs485` 모드는 실제 설정된 EV 상태 프레임이 수신될 때만 Discovery를 발행하도록 유지했습니다.

## 1.1.24-dev - 2026-04-25

### Added

- 엘리베이터 호출 방식을 `mqtt`(SOAP 외부 애드온 전달)와 `rs485`(애드온 직접 RS485 호출) 모드로 선택할 수 있게 했습니다.
- RS485 직접 호출에 사용할 호출 명령, 호출 ON 프레임, 호출중 프레임, 해제 프레임을 애드온 옵션으로 입력할 수 있게 했습니다.

### Changed

- 기본값은 기존 `CommaxEv` SOAP 애드온과 호환되도록 MQTT 전달 모드로 유지하며, RS485 명령은 `rs485` 모드에서만 전송합니다.
- 설정된 엘리베이터 상태 프레임이 수신되면 호출 상태를 `ON/OFF`로 갱신하고 ACK 매칭에도 활용합니다.

## 1.1.23-dev - 2026-04-25

### Changed

- 명령 즉시 배출은 다시 적용하지 않고, 우선순위 큐에서 같은 priority 명령이 들어온 순서대로 처리되도록 안정 정렬만 보강했습니다.

## 1.1.22-dev - 2026-04-25

### Fixed

- 직접 TCP 캡처로 확인한 `0x2A` 주차 장문 프레임의 `38..41` 바이트를 차량번호로 발행하도록 보정했습니다.
- `0x80` 차량번호 조각 프레임이 모두 패딩인 경우에는 기존처럼 무시하고, `0x2A` 프레임에서 주차 위치와 차량번호를 함께 갱신합니다.

## 1.1.21-dev - 2026-04-25

### Fixed

- 주차 차량번호 프레임에서 `6` 같은 단일 문자 조각은 무시하되, `1234`, `AR:1`, `A1`처럼 온전하게 해석되는 차량 표기는 발행하도록 오탐 방어 조건을 조정했습니다.

## 1.1.20-dev - 2026-04-25

### Changed

## 1.1.19-dev - 2026-04-25

### Added

- 복원된 상태 프레임은 `recovered_state_frame`으로 unknown 패킷 캡처에도 남겨 실제 오탐 여부를 추적할 수 있게 했습니다.

## 1.1.18-dev - 2026-04-25

### Changed

- 생활정보 `8F` 프레임은 완전 동일 반복값만 제외하고, 값이 변한 프레임은 날씨/미세먼지 분석을 위해 계속 수집되도록 유지했습니다.

- MQTT 명령 수신 시 HA 상태를 먼저 갱신하지 않고, 실제 월패드 ACK/상태 프레임 수신값으로만 상태가 바뀌도록 했습니다.

## 1.1.0 - 2026-04-25

이번 릴리스는 실사용 안정성과 Home Assistant 사용감을 크게 다듬은 기능 업데이트입니다.

### Added

- EW11 TCP 스트림 프레이밍을 추가해 쪼개지거나 붙어서 들어오는 패킷을 안정적으로 처리합니다.
- 원격 검침 `F7` 가변 길이 프레임을 지원합니다.
- 월간 검침 센서를 추가했습니다.
  - 이번달 전기 사용량
  - 이번달 수도 사용량
  - 이번달 온수 사용량
  - 이번달 난방 사용량
  - 이번달 가스 사용량
- 월 중간 설치 사용자를 위한 월간 사용량 보정 옵션을 추가했습니다.
- 명령 송신/수신 로그에 디바이스 종류, ID, 명령 종류, 응답 시간을 표시합니다.

### Changed

- 같은 디바이스에 빠르게 연속 명령이 들어올 때 최신 명령 중심으로 처리되도록 개선했습니다.
- 조명 밝기 명령 직후 Home Assistant가 보내는 중복 `ON` 명령을 억제합니다.
- 빠른 조작 시 ACK/STATE 매칭 정확도를 높여 불필요한 재전송 가능성을 줄였습니다.
- 명령 재전송 시점을 실제 송신 기준으로 조정해 빠른 연속 제어 안정성을 개선했습니다.
- 종료 처리에서 상태 저장과 availability publish가 더 안정적으로 수행되도록 정리했습니다.
- 검침, 공기질, 주차 센서에 값 성격에 맞는 아이콘을 지정했습니다.

### Fixed

- TCP 조각 수신으로 검침 프레임이 유실되거나 unknown packet으로 분리되던 문제를 수정했습니다.
- 짧은 공기질 패킷이 들어와도 파서가 죽지 않도록 방어 코드를 추가했습니다.
- 검침 장비가 분리된 환경에서도 원격 검침 프레임이 정상 파싱되도록 개선했습니다.

### Notes

- 월간 검침 센서는 애드온이 전체 누적값을 기준으로 내부 월초 기준값을 저장해 계산합니다.
